5 分ではじめる

スプレッドシートや GAS エディタが使えれば、ここから先は手順どおりで動きます。

1

ライブラリを追加する

Apps Script エディタの「ライブラリ」の ＋ を押して、この ID を貼り付けて「検索」→ 最新バージョンを選んで「追加」。

101aZZYpRRnr5AGkVIo8t_yo4kHb7xryLfq3w-HVPpQ4fX0Tkxv3UJyzc
2

Slack のトークンを用意する

Slack 側の準備はここが全部です(初回だけ・約 3 分):

  1. api.slack.com/apps で Create New App → From scratch(名前は自由、ワークスペースを選択)
  2. 左メニュー OAuth & Permissions → Bot Token Scopes に chat:write を追加(ファイルも送るなら files:write も)
  3. 同じページ上部の Install to Workspace → 許可
  4. 表示された Bot User OAuth Token(xoxb- で始まる)をコピー
  5. Apps Script の プロジェクトの設定 → スクリプト プロパティ に、プロパティ名 SLACK_ACCESS_TOKEN として保存
  6. 投稿したいチャンネルで /invite @アプリ名 して bot を招待(忘れると not_in_channel エラーになります)
3

3 行で投稿する

エディタに貼り付けて hello を実行。チャンネル ID はチャンネル名を右クリック →「チャンネル詳細を表示」の最下部にあります。

var token = PropertiesService.getScriptProperties().getProperty('SLACK_ACCESS_TOKEN');
var slack = GASlacker.methods(token);

function hello() {
  var res = slack.chat.postMessage({ channel: 'C0123456789', text: 'Hello from GAS!' });
  Logger.log(res.ok ? '投稿できました' : 'エラー: ' + res.error);
}

うまくいかないときはログの res.error を見てください。not_in_channel なら手順 2-6、invalid_auth ならトークンの保存名を確認します。

投稿できたら、もう完了です 🎉

ここまでできれば、あとは同じ書き方で Slack のほぼ全機能が使えます。 毎朝の自動通知にするなら、時計アイコンの「トリガー」から hello を時間主導型で登録するだけ。

できること

Slack Web API の 168 メソッドをそのままの名前で呼べます。よく使う形は examples/ にコピペ用スクリプトがあります。

シートから毎朝通知

時間トリガー + chat.postMessage。Block Kit で見やすい整形も。

返事をするボット

Events API を doPost で受けてスレッドに返信。

ファイルを送る

CSV やログを files.uploadV2 で 1 メソッド投稿。

🗂 全チャンネルを集計

slack.paginate() がページネーションを自動で辿ります。

📝 Canvas・リスト操作

canvases.* / slackLists.* など新しい API にも対応。

🧩 未対応でも呼べる

slack.call('メソッド名', params) でどの API でも。

// 例: シートの内容を毎朝 Slack へ(時間トリガーに設定)
function dailySummary() {
  var sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName('KPI');
  var rows = sheet.getRange(2, 1, sheet.getLastRow() - 1, 2).getValues();
  var lines = rows.map(function (r) { return '• ' + r[0] + ': *' + r[1] + '*'; });
  slack.chat.postMessage({ channel: 'C0123456789', text: lines.join('\n') });
}

168 メソッド、全部入り。

メッセージ・ファイル・リアクション・Canvas・通話まで、Slack 公式 SDK と同じ顔ぶれをそのままの名前で。

  • 全メソッド実在検証済み。存在しない・廃止された API は入っていません。
  • 週次パリティ監視。毎週 CI が公式 SDK と突き合わせ、新 API が出ると自動で Issue が立ちます。
  • トークンは手元だけ。通信先は Slack API のみ。あなたの Script プロパティにだけ保存されます。
  • TypeScript 対応。clasp 派には型定義を同梱。全メソッドが補完されます。

よくある質問

無料ですか?

はい。MIT ライセンスの OSS で、商用利用も自由です。Google Apps Script と Slack アプリも通常の用途では無料枠で動きます。

Node.js やビルド環境は必要?

不要です。ブラウザの Apps Script エディタだけで完結します(ライブラリ ID を追加するだけ)。

Incoming Webhook と何が違う?

Webhook は「決まったチャンネルへ投稿」しかできません。GASlacker はトークンで Web API を呼ぶので、返信・リアクション・ファイル・履歴取得・Canvas など Slack のほぼ全機能が使えます。

エラーが出たら?

すべてのメソッドは Slack のレスポンスをそのまま返します。res.ok が false のとき res.error に原因(例: channel_not_found)が入っています。解決しないときは Issues か Discussions へどうぞ(日本語 OK)。

Slack 公式のライブラリですか?

いいえ、コミュニティによる非公式 OSS です。Slack は Slack Technologies, LLC の商標です。
