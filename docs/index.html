Everyday automation

小さな自動化が、
毎日の仕事を軽くする。

Slack Web API をそのままの名前で呼べます。プログラミングが本業でなくても、まずは身近な作業から。

01 / MORNING UPDATE

シートの数字を、毎朝ちょうどよく届ける。

時間トリガーと chat.postMessage を組み合わせれば、KPI や当番表の共有を自動化できます。

02 / HELPFUL BOT

問い合わせに、そっと返事をするボット。

Events API を doPost で受けて、質問に応じた案内を同じスレッドへ返せます。

03 / FILE DELIVERY

できたレポートを、そのままチームへ。

CSV やログは files.uploadV2 で投稿。複雑なアップロード手順はライブラリが引き受けます。

Quick start

5 分で、
最初の投稿まで。

スプレッドシートや Apps Script エディタを使ったことがあれば、あとは手順どおりで動きます。

ブラウザだけで完結

ライブラリを追加する

Apps Script エディタの「ライブラリ」の ＋ を押し、この ID を貼り付けて「検索」→ 最新バージョンを選んで「追加」。 

101aZZYpRRnr5AGkVIo8t_yo4kHb7xryLfq3w-HVPpQ4fX0Tkxv3UJyzc

Slack のトークンを用意する

Slack 側の準備は初回だけ、約 3 分です。

  1. Slack API のアプリ管理画面で Create New App → From scratch
  2. OAuth & Permissions で chat:write を追加
  3. Install to Workspace からアプリをインストール
  4. xoxb- で始まる Bot User OAuth Token をコピー
  5. スクリプト プロパティへ SLACK_ACCESS_TOKEN の名前で保存
  6. 投稿先で /invite @アプリ名 を実行

3 行で投稿する

エディタへ貼り付けて hello を実行。チャンネル ID はチャンネル詳細の最下部で確認できます。 

var token = PropertiesService.getScriptProperties().getProperty('SLACK_ACCESS_TOKEN');
var slack = GASlacker.methods(token);

function hello() {
  var res = slack.chat.postMessage({ channel: 'C0123456789', text: 'Hello from GAS!' });
  Logger.log(res.ok ? '投稿できました' : 'エラー: ' + res.error);
}

not_in_channel なら bot の招待、invalid_auth ならトークンの保存名を確認してください。

Questions

よくある質問

無料ですか？

はい。MIT ライセンスの OSS で、商用利用も自由です。Google Apps Script と Slack アプリも通常の用途では無料枠で動きます。

Node.js やビルド環境は必要ですか？

不要です。ブラウザの Apps Script エディタだけで完結します。ライブラリ ID を追加してお使いください。

Incoming Webhook とは何が違いますか？

Webhook は決まったチャンネルへの投稿が中心です。GASlacker は Web API を呼ぶため、返信、リアクション、ファイル、履歴取得、Canvas など幅広い機能を使えます。

エラーが出たら？

Slack のレスポンスをそのまま返すため、res.error で原因を確認できます。解決しないときは Issues または Discussions へどうぞ。日本語で大丈夫です。

Slack 公式のライブラリですか？

いいえ、コミュニティによる非公式 OSS です。Slack は Slack Technologies, LLC の商標です。
